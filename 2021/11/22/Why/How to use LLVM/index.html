How to use LLVM

听闻了很久 llvm，看龙书的理论看了非常久，在今天又一次因为 IR。理论的碰壁，重新思考了下，为什么我想了解编译器：知其然，知其所以然。

可能吸收效率最高的方法是：知其然(代码), 知其所以然(理论)。实践部分大面积参考了 Buaa 的编译实验，理论部分参考了《编译原理》和《编译器设计》(Coorper)这两本书。

德尔非神谕的三句话：(随意加入)

  1. Know yourself 指的是人类要理解自己在时间和空间中的平行存在。可以参考通信中的时分和频分。
  2. Nothing in excess or Moderation 是指不要在一种时间/空间可能性上太过偏执，否则只能获得局部最优。
  3. Surety brings ruin 意味着一种确定态会导致其他态的坍塌，就像打开箱子之后薛定谔的猫。又因为没有一种可能性在无限的时间和空间里是绝对正确的，所以如果过分追求确定，到头来发现拯救世界的关键在于一开始的那只猫，那在打开箱子的surety就已经决定了ruin，之后所作的一切都没有意义了。

#5 LLVM IR

IR: Intermediate representation (中间表示)

5.1 IR

编译器工作过程可以分为三个部分：前端(front-end) - 中端(middle-end) - 后端(back-end)，每个部分的工作为：

  • 前端：源语言 –> IR ( Clang、Rustc 等)
  • 中端：优化 IR （ LLVM Optimizer ）
  • 后端：IR 翻译为目标语言，一般是目标体系结构的汇编，如MIPS/x86 （ LLVM static compiler ）
Fig 1: LLVM 工作流程

这种分工做法的好处是：

  1. 只需要在 IR 上做好优化，再翻译到汇编，这样能够在所有支持的体系结构上实现优化，减少了重复的工作量
  2. 举个例子，有 m 种语言，n 类编译器，如果直接将源语言翻译成目标语言，需要写 m*n 个编译器。而如果使用 IR 的中间形式 ，只需要 m+n 个编译器。

LLVM IR 三种等价的中间格式为：

  • 在内存中的编译中间语言 (无法通过文件的形式展示)
  • 硬盘上存储的二进制中间语言 (.bc)
  • 人可读的代码语言 (.ll)

5.3 IR 结构

参考资料：LLVM Programmer Manual

(1) 总体结构

  1. 基本单位为 module
  2. 一个 module 中有多个实体，如 function 和 global variable
  3. 一个 function define 中至少有一个 basicblock (基本块)
  4. 每个 basicblock 中有若干 instruction (指令)，并且都以 terminator instruction 结尾

(2) 函数定义与声明 (Define & Declare)

  1. 函数定义 define

函数定义的最简单语法：define i32 @func_name (i32 %a, i32% b, ...) { ... (body) },如基本的函数 main

加上参数列表

可以在参数列表后面加上属性来指导优化，如在 5.2 中 main.ll 后有 #0，其与 main.ll 靠后的 attributes #0 = ... 是对应的，被用来给函数加上特定解释属性，如是否能够被内联等。

(3) 基本块 (Basic Block)

基本块包含了若干个顺序执行的指令和一个终结指令的代码序列。

基本块内部没有控制流，基本块间实现跳转需要控制流。例如一个主函数内部没有跳转也没有函数调用，完全顺序执行的，那么这份代码中只有一个基本块。

如，在函数中间加入一个 if-else，则这个函数有4个基本块，如图

Fig 2:  函数内有 if-else 一共可以分为 4 个 basic block

(4) 指令 & 终结指令

一般指令 (Instruction) 指 LLVM IR 中的非分支指令，用于进行计算/访存，这些指令不会改变程序的控制流。 特殊的事，call 指令也是非分支指令，因为在使用 call 时，并不考虑被调用函数内部的情况，只关心当前传入的参数和被调用函数的返回值，不会改变当前程序的控制流。

终结指令 (Terminator Instruction)

一定位于基本块的末尾，终结指令决定了程序控制流的执行方向

  • ret 指令: 将程序控制流返回调用者
  • br 指令: 根据标识符 label，选择控制流方向

仅用于占位，不表示任何值/空间

  1. Integer type

i*, 表示 *bit 的长度，如 bool 只有 1-bit，则为 i1
